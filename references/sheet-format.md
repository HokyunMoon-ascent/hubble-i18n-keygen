# 34열 시트 포맷 & 컬럼 인덱스 맵

산출 CSV는 프로젝트의 마스터 시트(`aiOptimizer_sheet.csv`)와 **동일한 34열 헤더**를 미러링한다.
검수 전용 `<feature>_keys.csv` 에는 **이번에 새로 만든 키만** 담는다.

## 헤더 (34열)
```
위치1, 위치2, 작성일, 작업자, 메모, 삭제무방 및 비고, key1, key2, key3,
KR, JP, US, Column 13 … Column 34
```

## 사용 컬럼 인덱스 맵
`make_keys.py` 가 실제로 채우는 컬럼은 0~11 중 일부다. 나머지(작업자, key3, Column 13+)는 빈 칸으로 둔다.

| idx | 헤더            | 내용                                   |
|-----|-----------------|----------------------------------------|
| 0   | 위치1           | 기능 대분류 (예: `AI 옵티마이저`)       |
| 1   | 위치2           | 화면/섹션 (예: `4분면 차트`, `플랜 비교표`) |
| 2   | 작성일          | `YYYY-MM-DD`                            |
| 3   | 작업자          | (비움)                                 |
| 4   | 메모            | 키 의도·근거·node id 등 검수 메모        |
| 5   | 삭제무방 및 비고 | **중복 제외 사유** 기록란               |
| 6   | key1            | 네임스페이스 (`label`/`tooltip`/`plan`/`aiOptimizer` …) |
| 7   | key2            | 식별자 (camelCase)                      |
| 8   | key3            | (보통 비움)                             |
| 9   | KR              | 한국어 번역                            |
| 10  | JP              | 일본어 번역                            |
| 11  | US              | 영어 번역 (Mojito 현행 기준)            |

## 멀티라인 셀 (툴팁) 인용 규칙
툴팁 본문처럼 줄바꿈이 포함된 값은 셀 안에서 실제 개행으로 들어간다.
- spec(JSON)에서는 `\n` 으로 작성한다.
- `csv.writer` 가 개행·쉼표·따옴표를 포함한 셀을 자동으로 `"..."` 로 감싼다. 빈 줄(`\n\n`)도 그대로 보존된다.
- Google Sheets에 붙여넣으면 한 셀 안에서 여러 줄로 표시된다.

## 헤더 출처
헤더는 하드코딩하지 않고 `--template` 으로 받은 CSV의 첫 행에서 읽는다. 마스터 시트의 열 수가 바뀌어도 그대로 따라간다.
